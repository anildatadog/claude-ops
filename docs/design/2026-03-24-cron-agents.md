# Cron Agents Implementation Plan

> **Status:** ✅ Implemented 2026-03-24
> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create two autonomous cron agents — a daily morning brief and a weekly memory health check — that run without human involvement and surface their output automatically at session start.

**Architecture:** Two-step chain: `CronCreate → Claude agent → session-brief.sh → brief file → SessionStart reads file`. The cron Claude agent calls `session-brief.sh` (which reads local memory + runs worktree-health.sh) and then appends the Jira section via MCP. The brief file is written to `~/.claude/briefs/`. SessionStart reads the latest brief for today and prepends it to session context — it does NOT call `session-brief.sh` directly.

**Tech Stack:** Bash, `CronCreate` (Claude Code tool), `~/.claude/briefs/`, Atlassian MCP (Jira)

**Spec:** `~/.claude/docs/superpowers/specs/2026-03-24-ai-reliability-infrastructure-design.md` — System 3

## Task 2: Create `~/.claude/scripts/session-brief.sh`

**Files:**
- Create: `~/.claude/scripts/session-brief.sh`

This script is called at SessionStart (not directly by cron — cron invokes Claude which calls this). It reads local memory files and produces a markdown brief.

- [ ] **Step 1: Create the script**

Create `~/.claude/scripts/session-brief.sh`:

```bash
#!/usr/bin/env bash
# session-brief.sh — generate a morning brief from local memory files
# Output: ~/.claude/briefs/YYYY-MM-DD-HHmmss.md
# Called by: CronCreate Job 1 (daily 08:03) via Claude agent
# Also callable manually: bash ~/.claude/scripts/session-brief.sh
set -euo pipefail

BRIEFS_DIR="$HOME/.claude/briefs"
MEMORY_DIR="$HOME/.claude/projects/-Users-anilkumar-pappu-Documents-Git/memory"
TIMESTAMP=$(date +%Y-%m-%dT%H:%M:%S)
DATE=$(date +%Y-%m-%d)
BRIEF_FILE="$BRIEFS_DIR/${DATE}-$(date +%H%M%S).md"

mkdir -p "$BRIEFS_DIR"

# --- Header ---
cat > "$BRIEF_FILE" << EOF
# Morning Brief — $DATE

Generated: $TIMESTAMP
Source: local memory files (Jira section generated by Claude agent)

---

## Worktree Health

EOF

# --- Worktree health ---
bash "$HOME/.claude/scripts/worktree-health.sh" 2>/dev/null >> "$BRIEF_FILE" || echo "(worktree-health.sh unavailable)" >> "$BRIEF_FILE"

# --- RAG status from memory ---
echo "" >> "$BRIEF_FILE"
echo "---" >> "$BRIEF_FILE"
echo "" >> "$BRIEF_FILE"
echo "## Project RAG Status" >> "$BRIEF_FILE"
echo "" >> "$BRIEF_FILE"

for mem_file in \
    "$MEMORY_DIR/fca-session-log.md" \
    "$MEMORY_DIR/fca-cicd-progress.md" \
    "$MEMORY_DIR/fca-ust-progress.md"
do
    if [[ -f "$mem_file" ]]; then
        fname=$(basename "$mem_file")
        echo "### $fname" >> "$BRIEF_FILE"
        # Extract lines with RAG indicators or blockers
        grep -iE "🔴|🟡|🟢|blocker|blocked|pending|waiting|status:" "$mem_file" | head -20 >> "$BRIEF_FILE" || true
        echo "" >> "$BRIEF_FILE"
    fi
done

# --- Hook health summary ---
echo "---" >> "$BRIEF_FILE"
echo "" >> "$BRIEF_FILE"
echo "## Hook Health (previous session)" >> "$BRIEF_FILE"
echo "" >> "$BRIEF_FILE"

HOOK_LOG="$HOME/.claude/hook-log.txt"
if [[ -f "$HOOK_LOG" ]]; then
    YESTERDAY=$(date -v-1d +%Y-%m-%d 2>/dev/null || date -d "yesterday" +%Y-%m-%d 2>/dev/null)
    PREV_ENTRIES=$(grep "^$YESTERDAY" "$HOOK_LOG" 2>/dev/null | tail -20)
    if [[ -n "$PREV_ENTRIES" ]]; then
        echo "\`\`\`" >> "$BRIEF_FILE"
        echo "$PREV_ENTRIES" >> "$BRIEF_FILE"
        echo "\`\`\`" >> "$BRIEF_FILE"
    else
        echo "(No hook entries for $YESTERDAY)" >> "$BRIEF_FILE"
    fi
else
    echo "(hook-log.txt not found)" >> "$BRIEF_FILE"
fi

echo "" >> "$BRIEF_FILE"
echo "---" >> "$BRIEF_FILE"
echo "" >> "$BRIEF_FILE"
echo "## Jira" >> "$BRIEF_FILE"
echo "" >> "$BRIEF_FILE"
echo "_Jira section populated by Claude agent (requires Atlassian MCP)_" >> "$BRIEF_FILE"
echo "" >> "$BRIEF_FILE"

echo "$BRIEF_FILE"
```

## Task 3: Create `~/.claude/scripts/memory-health.sh`

**Files:**
- Create: `~/.claude/scripts/memory-health.sh`

- [ ] **Step 1: Create the memory health script**

Create `~/.claude/scripts/memory-health.sh`:

```bash
#!/usr/bin/env bash
# memory-health.sh — weekly memory health check
# Output: ~/.claude/briefs/memory-health-YYYY-MM-DD.md
# Called by: CronCreate Job 2 (weekly Monday 08:07) via Claude agent
set -euo pipefail

BRIEFS_DIR="$HOME/.claude/briefs"
MEMORY_DIR="$HOME/.claude/projects/-Users-anilkumar-pappu-Documents-Git/memory"
DATE=$(date +%Y-%m-%d)
OUTPUT="$BRIEFS_DIR/memory-health-$DATE.md"
HOOK_LOG="$HOME/.claude/hook-log.txt"
NOW_EPOCH=$(date +%s)
THIRTY_DAYS_SECS=$((30 * 24 * 3600))
SEVEN_DAYS_SECS=$((7 * 24 * 3600))

mkdir -p "$BRIEFS_DIR"

cat > "$OUTPUT" << EOF
# Memory Health Check — $DATE

---

## Stale Index Entries (not updated in 30+ days)

EOF

# Check confluence-gdrive-index.md for stale timestamps
if [[ -f "$MEMORY_DIR/confluence-gdrive-index.md" ]]; then
    echo "### confluence-gdrive-index.md" >> "$OUTPUT"
    # Extract lines with dates and flag those older than 30 days
    while IFS= read -r line; do
        if echo "$line" | grep -qE "[0-9]{4}-[0-9]{2}-[0-9]{2}"; then
            entry_date=$(echo "$line" | grep -oE "[0-9]{4}-[0-9]{2}-[0-9]{2}" | tail -1)
            if [[ -n "$entry_date" ]]; then
                entry_epoch=$(date -j -f "%Y-%m-%d" "$entry_date" +%s 2>/dev/null || date -d "$entry_date" +%s 2>/dev/null || echo 0)
                age=$(( NOW_EPOCH - entry_epoch ))
                if (( age > THIRTY_DAYS_SECS )); then
                    echo "⚠️  STALE ($entry_date): $line" >> "$OUTPUT"
                fi
            fi
        fi
    done < "$MEMORY_DIR/confluence-gdrive-index.md"
    echo "" >> "$OUTPUT"
fi

# --- Stop hook silence detection ---
echo "---" >> "$OUTPUT"
echo "" >> "$OUTPUT"
echo "## Hook Health — Stop Hook Silence Check" >> "$OUTPUT"
echo "" >> "$OUTPUT"

if [[ -f "$HOOK_LOG" ]]; then
    SEVEN_DAYS_AGO=$(date -v-7d +%Y-%m-%d 2>/dev/null || date -d "7 days ago" +%Y-%m-%d 2>/dev/null)
    RECENT_ENTRIES=$(grep -c "STOP" "$HOOK_LOG" 2>/dev/null || echo 0)
    RECENT_BLOCKS=$(grep "STOP BLOCK" "$HOOK_LOG" 2>/dev/null | wc -l | tr -d ' ')
    TOTAL_ENTRIES=$(wc -l < "$HOOK_LOG" | tr -d ' ')

    echo "Total hook log entries: $TOTAL_ENTRIES" >> "$OUTPUT"
    echo "STOP entries (all time): $RECENT_ENTRIES" >> "$OUTPUT"
    echo "STOP BLOCK entries (all time): $RECENT_BLOCKS" >> "$OUTPUT"
    echo "" >> "$OUTPUT"

    # Flag anomalous silence: >10 entries total, 0 blocks in last week with git activity
    if (( TOTAL_ENTRIES > 10 && RECENT_BLOCKS == 0 )); then
        echo "⚠️  ANOMALY: >10 hook log entries but zero STOP BLOCKs — Stop hook may not be firing" >> "$OUTPUT"
        echo "Check: is stop-hook.sh present and registered in settings.json?" >> "$OUTPUT"
    else
        echo "✅ Stop hook appears to be firing normally" >> "$OUTPUT"
    fi
else
    echo "⚠️  hook-log.txt not found at $HOOK_LOG" >> "$OUTPUT"
fi

echo "" >> "$OUTPUT"
echo "---" >> "$OUTPUT"
echo "" >> "$OUTPUT"
echo "## Repo Inventory Cross-Reference" >> "$OUTPUT"
echo "" >> "$OUTPUT"

REPO_INVENTORY="$MEMORY_DIR/repo_inventory.md"
GIT_DIR="$HOME/Documents/Git"

if [[ -f "$REPO_INVENTORY" ]]; then
    # Extract repo names from inventory (lines like "**Local path:** .../Git/<name>" or lines under "### <name>")
    INVENTORY_REPOS=$(grep -oE "Documents/Git/[a-zA-Z0-9_-]+" "$REPO_INVENTORY" | sed 's|Documents/Git/||' | sort -u)
    ACTUAL_DIRS=$(ls -d "$GIT_DIR"/*/  2>/dev/null | xargs -I{} basename {} | sort -u)

    echo "### Repos in inventory but not on disk" >> "$OUTPUT"
    while IFS= read -r repo; do
        if [[ -n "$repo" ]] && ! echo "$ACTUAL_DIRS" | grep -qx "$repo"; then
            echo "⚠️  MISSING ON DISK: $repo" >> "$OUTPUT"
        fi
    done <<< "$INVENTORY_REPOS"

    echo "" >> "$OUTPUT"
    echo "### Repos on disk not in inventory" >> "$OUTPUT"
    while IFS= read -r dir; do
        if [[ -n "$dir" ]] && ! grep -q "$dir" "$REPO_INVENTORY"; then
            echo "⚠️  NOT IN INVENTORY: $dir" >> "$OUTPUT"
        fi
    done <<< "$ACTUAL_DIRS"

    echo "" >> "$OUTPUT"
    # Check for recent commits in tracked repos with no inventory update in 30 days
    echo "### Repos with recent commits (check inventory is current)" >> "$OUTPUT"
    SEVEN_DAYS_AGO_DATE=$(date -v-7d +%Y-%m-%d 2>/dev/null || date -d "7 days ago" +%Y-%m-%d 2>/dev/null)
    while IFS= read -r repo; do
        repo_path="$GIT_DIR/$repo"
        if [[ -d "$repo_path/.git" ]]; then
            recent=$(git -C "$repo_path" log --since="$SEVEN_DAYS_AGO_DATE" --oneline 2>/dev/null | wc -l | tr -d ' ')
            if (( recent > 0 )); then
                echo "📝  $repo — $recent commits in last 7 days (verify inventory entry is current)" >> "$OUTPUT"
            fi
        fi
    done <<< "$INVENTORY_REPOS"
else
    echo "⚠️  repo_inventory.md not found at $REPO_INVENTORY" >> "$OUTPUT"
fi

echo "" >> "$OUTPUT"

echo "$OUTPUT"
```

**Approach note:** The spec says to extend `worktree-health.sh`. This plan uses an inline command extension in `settings.json` instead, because: (a) the brief-reading logic is SessionStart-specific (not relevant to standalone health checks), and (b) it avoids making `worktree-health.sh` depend on a `briefs/` directory that may not exist in all environments. - [ ] **Step 2: Update SessionStart hook command in settings.json**

The new command should:
1. Run worktree-health.sh as before (already includes hook-health-check.sh)
2. After health output, check for today's brief and print it if found

New command string (replace current SessionStart command):
```
bash -c 'echo "--- Worktree Health ---"; ~/.claude/scripts/worktree-health.sh 2>/dev/null; echo "------------------------"; BRIEF=$(ls -t ~/.claude/briefs/'"$(date +%Y-%m-%d)"'-*.md 2>/dev/null | head -1); if [[ -n "$BRIEF" ]]; then echo "--- Morning Brief ---"; cat "$BRIEF"; echo "---------------------"; fi'
```

**Important:** The `date +%Y-%m-%d` must be evaluated at hook runtime, not at settings.json write time. Use `$(date +%Y-%m-%d)` — but since this is a bash -c string, the shell will expand it correctly at execution time.

Correct approach — write a Python script to update settings.json:

```python
import json

settings_path = "/Users/anilkumar.pappu/.claude/settings.json"
with open(settings_path) as f:
    d = json.load(f)

new_cmd = (
    "bash -c '"
    "echo \"--- Worktree Health ---\"; "
    "~/.claude/scripts/worktree-health.sh 2>/dev/null; "
    "echo \"------------------------\"; "
    "TODAY=$(date +%Y-%m-%d); "
    "BRIEF=$(ls -t ~/.claude/briefs/${TODAY}-*.md 2>/dev/null | head -1); "
    "if [[ -n \"$BRIEF\" ]]; then "
    "echo \"--- Morning Brief ---\"; "
    "cat \"$BRIEF\"; "
    "echo \"---------------------\"; "
    "fi'"
)

d["hooks"]["SessionStart"][0]["hooks"][0]["command"] = new_cmd

with open(settings_path, "w") as f:
    json.dump(d, f, indent=2)
print("Updated SessionStart hook")
```
